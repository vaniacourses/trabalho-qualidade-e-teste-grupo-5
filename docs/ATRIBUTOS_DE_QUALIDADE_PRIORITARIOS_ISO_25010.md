# Atributos Prioritários da ISO/IEC 25010 para o Sistema MedAlerta

## 1. Confiabilidade (Reliability) — Prioridade Muito Alta

O sistema existe para emitir alertas nos horários corretos. Se ele falhar, travar ou deixar de emitir notificações, o usuário pode esquecer de tomar um medicamento importante.

### Possíveis medidas

* Percentual de alertas emitidos corretamente.
* Taxa de falhas durante a execução.
* Disponibilidade do sistema.
* Tempo médio entre falhas (MTBF).

### Exemplo de metas

* 99,9% dos alertas devem ser disparados corretamente.
* Disponibilidade superior a 99%.

---

## 2. Usabilidade (Usability) — Prioridade Muito Alta

O público-alvo são pessoas idosas, que podem possuir dificuldades visuais, cognitivas ou pouca familiaridade com tecnologia. Mesmo que o sistema funcione perfeitamente, ele não será útil se os usuários tiverem dificuldade para cadastrar medicamentos ou interpretar os alertas.

### Possíveis medidas

* Tempo necessário para cadastrar um medicamento.
* Taxa de erros cometidos pelos usuários.
* Grau de satisfação dos usuários.
* Percentual de usuários que conseguem concluir tarefas sem auxílio.

### Exemplo de metas

* 90% dos usuários devem conseguir cadastrar um medicamento sem ajuda.
* Cadastro concluído em menos de 3 minutos.

---

## 3. Segurança (Security) — Prioridade Alta

O sistema armazena dados pessoais e informações relacionadas à saúde dos usuários. Dados como medicamentos utilizados, endereço, telefone e e-mail são informações sensíveis e devem ser protegidas contra acesso indevido.

### Possíveis medidas

* Quantidade de vulnerabilidades encontradas.
* Percentual de acessos autenticados.
* Taxa de tentativas de acesso indevido bloqueadas.

### Exemplo de metas

* 100% das áreas restritas exigem autenticação.
* Nenhuma vulnerabilidade crítica identificada.
